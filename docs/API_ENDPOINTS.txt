API BASE
- Base URL: /api/v1
- Content-Type default: application/json
- Auth endpoint protected: Authorization: Bearer <token>

RESPONSE FORMAT (umum)
- Success:
  {
    "success": true,
    "message": "OK",
    "data": ...,
    "meta": ... (opsional, untuk pagination)
  }
- Error:
  {
    "success": false,
    "message": "Pesan error",
    "errors": ... (opsional)
  }

!!! CATATAN PENTING (UPDATE) !!!
Semua path gambar sekarang adalah URL ABSOLUT (lengkap dengan http/https).
Flutter tidak perlu lagi menambahkan Base URL secara manual ke path gambar.
Gunakan langsung: Image.network(product.image)

==================================================
PUBLIC ENDPOINTS (tanpa token)
==================================================

HOME
1. GET /api/v1/home
   Query (opsional):
   - limit (int, default 10, min 1, max 50)
   Return data:
   - banners[]
   - categories[] (maksimum 20 data)
   - flashsale_products[]
   - terlaris_products[]
   - terbaru_products[]
   - promo_products[] (alias flashsale)
   - popular_products[] (alias terlaris)
   - latest_products[] (alias terbaru)

2. GET /api/v1/home/flashsale
   Query (opsional): limit (int, default 10, min 1, max 50)
   Return data: products[]

3. GET /api/v1/home/terlaris
   Query (opsional): limit (int, default 10, min 1, max 50)
   Return data: products[]

4. GET /api/v1/home/terbaru
   Query (opsional): limit (int, default 10, min 1, max 50)
   Return data: products[]

BANNERS
5. GET /api/v1/banners
   Query (opsional): limit (int, default 10, min 1, max 50)
   Return data: banners[] aktif

CATEGORIES
6. GET /api/v1/categories
   Query (opsional):
   - q (string, cari nama kategori)
   - per_page (int, default 20, min 1, max 50)
   Return data:
   - array kategori (id, name, slug, description, products_count)
   - meta pagination

7. GET /api/v1/categories/{slug}
   Query (opsional):
   - sort (string: newest | price_asc | price_desc | sold | rating)
   Return data:
   - category {}
   - products[]
   - meta pagination

PRODUCTS
8. GET /api/v1/products
   Query (opsional):
   - q (string)
   - category_id (int)
   - category_slug (string)
   - sort (string: newest | price_asc | price_desc | sold | rating)
   Return data: products[], meta pagination

9. GET /api/v1/products/{slug}
   Return data:
   - product {}
   - related_products[]

10. GET /api/v1/products/{slug}/related
    Return data: products[]

SEARCH (GLOBAL)
11. GET /api/v1/search?q={query}
    - Minimal 2 karakter query.
    - Return data: { products: [], blogs: [] }

BLOGS
12. GET /api/v1/blogs
    Query (opsional): 
    - per_page (int)
    - q (string, search title/content)
    - category_id (int)
    - category_slug (string)
    Return data: blogs[], meta pagination

13. GET /api/v1/blog-categories
    Return data: array kategori blog (id, name, slug, blogs_count)

14. GET /api/v1/blogs/{slug}
    Return data: blog {}

15. GET /api/v1/blogs/{slug}/related
    Query (opsional): limit (int)
    Return data: blogs[]

CONTOH PENERAPAN DI FLUTTER

1. MODEL PRODUCT (dart)
class Product {
  final int id;
  final String name;
  final String image; // URL Absolut dari API
  final double price;

  Product({required this.id, required this.name, required this.image, required this.price});

  factory Product.fromJson(Map<String, dynamic> json) {
    return Product(
      id: json['id'],
      name: json['name'],
      image: json['image'] ?? '', // Langsung pakai, sudah absolut
      price: (json['price'] as num).toDouble(),
    );
  }
}

2. HANDLING IMAGE DI UI
Semua field gambar (image, icon, cover_image, url) sudah siap pakai:

// Di Flutter Widget
CachedNetworkImage(
  imageUrl: product.image, // Nilai dari API: "http://domain.com/storage/products/xxx.webp"
  placeholder: (context, url) => CircularProgressIndicator(),
  errorWidget: (context, url, error) => Icon(Icons.broken_image),
)
